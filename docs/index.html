Claude Code that never sleeps.

Claude Code in the browser is great — until your laptop sleeps, your network blinks, or the session times out and your context is gone. pager runs claude --remote-control inside a persistent tmux on your own always-on Linux box or Mac. No timeouts. No "session expired." No "please reconnect." Same session is alive when you wake up tomorrow, when you switch from laptop to phone, when the rig reboots.

Open the URL on any phone signed into your claude.ai account → you're inside the same shell, the same context, the same conversation. No SSH tunnel. No port forward. No relay you don't already trust.

Linux + macOS $curl -fsSL https://raw.githubusercontent.com/jawwadzafar/pager/main/install.sh | sh
Windows (alpha) PS>irm https://raw.githubusercontent.com/jawwadzafar/pager/main/install.ps1 | iex

One command per platform. Clones to ~/.pager (Linux/Mac) or %USERPROFILE%\.pager (Windows), installs deps, and registers autostart so the session survives reboots. Windows is pure native PowerShell — no WSL needed.

What pager gives you

One-command install

Idempotent bootstrap.sh (Linux, apt) or macos/bootstrap.sh (macOS, brew). Prereqs, env, PATH, Claude Code trust flag, service or LaunchAgent, watchdog. Re-run anytime; only fixes what's missing.

Linux + macOS + Windows

Native on all three. Linux uses systemd --user with loginctl enable-linger for boot-time start. macOS uses LaunchAgents at login (Tahoe 26 / Sequoia 15, Apple Silicon and Intel). Windows (alpha) uses a logon-triggered Scheduled Task with battery-resilient settings (Win10/11, PowerShell 5.1+, no WSL). One bin/pager (bash) for Linux/Mac, one bin/pager.ps1 for Windows — identical command surface.

Survives reboots

A user-level service brings the session back automatically. On Linux that's linger-enabled boot-time start; on macOS it's the login-time LaunchAgent. Your phone URL stays current either way.

Self-healing watchdog

A timer ticks every ~70s (pager-watch.timer on Linux, com.pager.watch LaunchAgent on macOS), logs session state to logs/watch.csv, and restarts a dead session automatically.

One-command diagnosis

pager doctor walks the full install — deps, PATH shadowing, .env perms, trust flag, OS-aware service checks (systemctl or launchctl), watchdog freshness — with a fix hint on every failure.

Multiple sessions

pager start work spawns a second independent session with its own URL and log. Separate context, separate window on your phone.

Inventory-driven SSH helper

Small actions/ + inventory.yaml layer lets the always-on Claude operate your fleet from inside the session. Credentials in .env, never echoed.

Know it's working: pager doctor

Run after install, after a reboot, when something looks off. Exits non-zero if anything's broken, with a fix hint inline.

Install

Linux + macOS $curl -fsSL https://raw.githubusercontent.com/jawwadzafar/pager/main/install.sh | sh
Windows (alpha) PS>irm https://raw.githubusercontent.com/jawwadzafar/pager/main/install.ps1 | iex

One command per platform. Requires only git (or winget on Windows, which the installer can pull Git through). Clones into ~/.pager on Linux/Mac or %USERPROFILE%\.pager on Windows, then runs the right platform bootstrap. Idempotent — re-run any time to update.

Linux Debian, Ubuntu, any modern apt box

Installs tmux, sshpass, python3-yaml via apt. Registers pager.service + pager-watch.timer. Enables linger so it starts at boot before any login.

macOS Tahoe 26 / Sequoia 15, Apple Silicon or Intel

Installs Homebrew (if missing), then brew install tmux libyaml python@3.13 + optional hudochenkov/sshpass. Registers com.pager.agent LaunchAgent. Starts at login. macOS notes →

Windows (alpha) Win10 / Win11, PowerShell 5.1+ — no WSL

Installs Git + OpenSSH + Python via winget. Wires $PROFILE, pre-trusts $env:USERPROFILE in ~/.claude.json, registers a pager Scheduled Task at logon. pager attach becomes pager logs (read-only tail); everything else (start/stop/status/url/trust/autostart/uninstall) is identical to Linux/Mac.

After install

  1. Open a new shell — source ~/.bashrc on Linux, source ~/.zprofile on macOS, or a fresh PowerShell window on Windows so $PROFILE reloads — then:

    pager doctor # everything green? pager url # phone URL

  2. Open that URL on a phone signed into the same claude.ai account. Done.

Or, the manual way

Linux & macOS:

$ git clone https://github.com/jawwadzafar/pager.git ~/.pager
$ cd ~/.pager
$ ./install.sh                # same auto-detect
$ ./linux/bootstrap.sh        # or, explicitly
$ ./macos/bootstrap.sh

Windows (PowerShell):

PS> git clone https://github.com/jawwadzafar/pager.git $env:USERPROFILE\.pager
PS> cd $env:USERPROFILE\.pager
PS> .\windows\bootstrap.ps1     # add -NoAutostart to skip the Scheduled Task

Uninstall

Linux & macOS:

$ pager uninstall          # stops the service/LaunchAgent, removes the symlink + shell-rc lines
$ rm -rf ~/.pager          # optional: final wipe of the repo + logs + .env

Windows:

PS> pager uninstall                                       # Scheduled Task + $PROFILE block
PS> Remove-Item -Recurse -Force $env:USERPROFILE\.pager   # optional: final wipe

pager uninstall is non-destructive: tears down system integration but never touches .env or the repo. Add -y to skip the prompt.
