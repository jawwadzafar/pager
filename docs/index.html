The 60-second phone walkthrough

  ┌─────────────────┐     ┌─────────────────┐     ┌────────────────────┐
  │  Linux box      │     │  claude.ai      │     │  Your phone        │
  │  (this repo)    │ ──► │  relay          │ ──► │  claude.ai/code    │
  │                 │     │                 │     │                    │
  │  tmux: claude   │     │  session_…      │     │  attached!         │
  │  --remote-      │     │                 │     │                    │
  │  control        │     │                 │     │                    │
  └─────────────────┘     └─────────────────┘     └────────────────────┘
  1. Install on the box (once). git clone https://github.com/jawwadzafar/pager.git ~/pager && ~/pager/bootstrap.sh Installs deps, wires the shell, enables linger, starts pager.service. A tmux session named claude is already running claude --remote-control.
  2. Get the URL. pager url # → https://claude.ai/code/session_01XXXXXXXXXXXXXX
  3. Open that URL on your phone (signed into the same claude.ai account). You're now talking to the same Claude session that lives on your box. Type on phone, type on terminal — both stream to the same conversation.
  4. Reboot the box. When it comes back, the session is already up. pager url for the new link.

What pager gives you

One-command install

bootstrap.sh is idempotent — apt prereqs, env, PATH, Claude Code trust flag, systemd unit, linger, watchdog. Re-run anytime; only fixes what's missing.

Survives reboots

A user-level systemd service with linger enabled brings the session back at boot, before any login. Your phone URL stays current.

Self-healing watchdog

pager-watch.timer ticks every 60s, logs session state to logs/watch.csv, and restarts a dead session automatically.

One-command diagnosis

pager doctor walks the full install — deps, PATH shadowing, .env perms, trust flag, services, linger, watchdog freshness — with a fix hint on every failure.

Multiple sessions

pager start work spawns a second independent session with its own URL and log. Separate context, separate window on your phone.

Inventory-driven SSH helper

Small actions/ + inventory.yaml layer lets the always-on Claude operate your fleet from inside the session. Credentials in .env, never echoed.

Know it's working: pager doctor

Run after install, after a reboot, when something looks off. Exits non-zero if anything's broken, with a fix hint inline.

Install

  1. On any Ubuntu/Debian/modern-Linux box:

    git clone https://github.com/jawwadzafar/pager.git ~/pager && ~/pager/bootstrap.sh

  2. Open a new shell (or source ~/.bashrc), then:

    pager doctor # everything green? pager url # phone URL

  3. Open that URL on a phone signed into the same claude.ai account. Done.

Full docs, CLI reference, and design notes live in the repo README.
